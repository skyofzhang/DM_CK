## r38 关键教训（超 r37 教训累积）

1. **半成品延续模式第 8 轮再现**：r37 commit message 标"BuildVoteUI.SendPropose r37 GAP-C37-02 真闭环"，但 r38 grep 验证发现 0 处真实调用源 — 公开方法定义 ≠ 真闭环。**每个 commit message 标"闭环"必须满足"方法定义 + 真实调用源 ≥ 1 处 + 完整调用链可达"三条件**。
2. **Agent 误报率高于真 GAP 率（~60% 误报 vs r37 几乎 100% 真）**：本轮 91 raw → ~25 真，误报率 60%+。原因：Agent Explore 模式快速侦察未做"grep 确认实际使用 + Read 完整方法验证消费"二次验证。**PM 系统级深挖必须 grep + Read 配对验证每个 MAJOR 候选**，否则"修复"假 GAP 浪费时间。
3. **闭环规模趋稳信号（r38 6 项 < r37 25 项 < r36 20 项 < r35 5 项 < r25 36 项）**：r35-r38 闭环规模整体下降趋势（除 r37 大爆发外），意味着代码层趋于稳定，剩余多为 doc 同步 + 边界情况 + 未来功能扩展。**r38 真 GAP 数量第二少（仅高于 r35 5 项）**——可宣告代码层趋于"接近完成"。
4. **集中决策（接力清理模式终结）**：SFX 孤儿在 r28-r37 接力清理但未集中决策（多轮加部分注释不全），r38 集中决策保留 + 统一 [Legacy] 标签 + 加 'sfx_legacy_purge' 搜索 tag — **未来类似分散问题应一次性集中决策避免接力浪费**。
5. **Agent 误报澄清的价值**：本轮 60 项误报澄清不是浪费，反而是"r38 验证 r35-r37 多轮修复完整性"的一次性快照。这些误报澄清成果记入 v38 美术清单，作为未来 audit 的"已确认实装清单"，避免 r39+ Agent 重复报告同一项。
