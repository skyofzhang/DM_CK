# SurvivalLiveRankingUI_GameUIPanel UI 策划案

> Prefab：`Assets/Prefabs/UI/Panels/SurvivalLiveRankingUI_GameUIPanel.prefab`  
> 用途：游戏进行中综合 HUD / 直播互动战斗界面  
> 主效果图规格：1080 x 1920 竖屏  
> 日期：2026-05-07

## 1. 界面定位

`SurvivalLiveRankingUI_GameUIPanel` 不只是一个排行榜窗口，它实际承担的是“极地生存”玩法进入 Running 状态后的全屏战斗 HUD。画面中间需要留给游戏主体、角色、建筑、怪物、特效和摄像机表现，UI 负责在四周提供直播互动信息、榜单、资源贡献、礼物反馈、事件提示和建造投票。

这套界面要做成“直播观众一眼看懂、主播长时间开播不累、玩家送礼/发弹幕有反馈”的风格。它不是大厅页，也不是营销页，不能做成巨大标题页；第一屏就是可用的游戏内 HUD。

## 2. 设计目标

1. **核心状态清楚**：玩家必须立刻知道现在是白天/夜晚、资源是否危险、城防/基地是否安全、倒计时还有多久。
2. **贡献反馈强**：观众送礼、上榜、被聚光灯高亮时，需要有明显但不遮挡游戏主体的反馈。
3. **榜单可读**：竖屏下守护者榜 Top5 要稳定、清晰，名字、名次、贡献值不要挤压。
4. **直播信息不断流**：弹幕动态、跑马灯、VIP 公告、加入提示、礼物动画要形成“直播热度”，但层级要分明。
5. **便于 prefab 调整**：每个功能区有明确边界，后续可以单独改 `LiveRankingPanel`、`ResourceRankPanel`、`BarragePanel`、`GiftAnimation` 等区域。

## 4. 推荐画面结构

### 4.1 顶部状态区

位置：屏幕顶部 0-110px 范围。  
目标：一眼看到当前局势。

包含内容：

- 局数/天数：`第 3 天`
- 阶段：`白天搜集` / `夜晚防守` / `暴风雪来袭`
- 倒计时：大数字，优先级高
- 城门/基地血量：红绿渐变条
- 食物、煤炭、矿石、温度/热量资源 chip
- 赛季或活动入口：尺寸小，不要抢主状态栏

视觉建议：

- 顶栏使用深蓝黑半透明底，带冰晶细边。
- 白天状态用冰蓝和白色；夜晚/危险用琥珀和红色脉冲。
- 资源图标必须比文字更容易识别：食物暖黄、煤炭灰黑、矿石银蓝、温度红橙。

### 4.2 游戏主体右上悬浮守护者榜

位置：游戏主体画面右上角悬浮位，也就是你截图绿框位置，建议参考 1080x1920 画布中的 `x=850 y=288 w=174 h=350`。  
节点：`LiveRankingPanel`。  
标题：`守护者榜`。

内容结构：

- 标题条：图标 + `守护者榜` + 小字 `本局贡献`
- Top5 行：名次、短玩家名、缩写贡献值
- 第一名需要更强视觉：金色徽章、皇冠/星形符号、轻微发光
- 2-3 名使用银蓝/铜金弱化层级
- 4-5 名保持普通深色行

脚本行为必须保留：

- `SurvivalLiveRankingUI` 会在 Running 状态显示面板。
- 排名数据最多显示 5 行。
- 行内 TMP 数量可能是 1、2 或 3 个，推荐效果图按 3 列表现：`#1` / `玩家名` / `12,340`。
- 玩家名会截断到 6 个字符，效果图里也不要设计超长文本。
- 聚光灯效果会把玩家名前缀变成金色 `★`，效果图要表现这个状态。
- 排名变化时行背景会短暂金色闪烁，效果图可做一个“正在升榜”的状态。

行状态设计：

| 状态 | 视觉 |
| --- | --- |
| 普通 | 深蓝半透明行底，白字，贡献值冰蓝 |
| Top1 | 金色徽章，名字略大，贡献值金白发光 |
| 自己/重点玩家 | 左侧有细高亮条，可用极光青 |
| 聚光灯 | 金色外发光、`★` 前缀、行底有柔和金色扫光 |
| 排名变化 | 0.3 秒金色闪光，可在效果图中表现为一条斜向光扫过 |
| 空数据 | 显示 `#-`、`等待守护者`、`0` 或直接隐藏空行，不要出现破碎占位 |

### 4.3 游戏主体左上悬浮资源贡献榜

位置：游戏主体画面左上角悬浮位，也就是你截图绿框位置，建议参考 1080x1920 画布中的 `x=56 y=288 w=174 h=350`。  
节点：`ResourceRankPanel`。  
标题列：`食物贡献`、`煤炭贡献`、`矿石贡献`。

内容结构：

- 三组 Top3，竖向堆叠：食物、煤炭、矿石。
- 每行格式：`1. 玩家名  1,280`。
- 暂无数据：`1. —`，但效果图建议做成灰蓝色弱占位。

视觉建议：

- 三组不要做太厚重，避免遮挡游戏主体。
- 列标题使用资源图标 + 中文标题。
- 食物列暖黄，煤炭列冷灰，矿石列银蓝。
- 行背景可只有浅描边，不要每行都做厚卡片。

### 4.4 底部弹幕和指令区

位置：游戏主体安全区下方，建议参考 1080x1920 画布中的 `x=40 y=1324 w=1000 h=132`。  
节点：`BarragePanel`、`HintText`、`BarrageDivider`。

当前 prefab 文案里存在：

- `弹幕动态`
- `发送 1 加入冰蓝队 | ...`
- `送礼物推动雪球，帮阵营赢得胜利！`
- `雪球碰到边界即判定获胜，加油！`

策划建议：

- 如果当前玩法已经从“推雪球阵营”切到“极地生存守城”，效果图不要继续强化雪球文案，可以替换成生存玩法文案。
- 弹幕区要像直播信息流，不要像聊天软件大窗口。
- 一行消息最多 32-40 中文字符，超出滚动或淡出。
- 重要观众行为可以加彩色标签：`送礼`、`加入守护`、`修复城门`、`补充煤炭`。

推荐文案：

- `发送 食物 帮队伍补给`
- `发送 煤炭 提升炉火`
- `发送 修门 守住城门`
- `送礼触发守护技能`
- `最高贡献者将登上守护者榜`

### 4.5 礼物、VIP、事件反馈区

位置：优先中上、右中、底部中，不要压住玩家核心操作区。  
节点：`GiftAnimation`、`VIPAnnouncement`、`EventTriggeredToast`、`SupporterMarquee` 等。

设计重点：

- 礼物动画：短、亮、强反馈，0.8-1.5 秒内完成主体爆发。
- 礼物贴纸说明区：不能只做成小按钮入口，需要有足够面积展示礼物图标、名称、档位/消耗和游戏内效果描述。施工稿建议放在底部右侧 `x=384 y=1480 w=656 h=236`，默认常驻，符合抖音礼物贴纸展示要求。
- 当前项目礼物描述可先按 `GiftAnimationUI.GiftInfo` 施工：仙女棒=效率+5%、能力药丸=全员效率+50%、甜甜圈=城门+200/食物+100、能量电池=炉温+30℃/效率+30%、神秘空投=超级补给、爱的爆炸=AOE伤害/矿工全满/城门+200、魔法糖=效率+10%、超能暴射=全场伤害+50%。
- `GiftIconBar` 可以作为快捷入口保留，但要并入贴纸说明区；具体描述优先复用项目里 `StickerPanelUI` / `GiftAnimationUI` 已有的礼物说明数据。
- VIP 公告：金色/紫金/极光色，但不要长期停留。
- 事件触发：危险事件用红橙，奖励事件用金色，协作完成用绿色/冰蓝。
- 跑马灯：只做一条主要高优先级跑马灯，不要顶部多条信息互相抢。

## 5. 推荐视觉风格

主题关键词：

- 极地生存
- 冰雪堡垒
- 直播互动
- 深夜防守
- 冰晶玻璃 HUD
- 高贡献玩家高光

基础色彩：

| 用途 | 色值 | 说明 |
| --- | --- | --- |
| 主背景遮罩 | `#07111F` / 60-75% | 深冷色，保证文字可读 |
| 面板底 | `#10263A` / 70-85% | 冰蓝黑半透明 |
| 主高亮 | `#65D6FF` | 冰晶蓝 |
| 强高亮 | `#FFD66B` | Top1、聚光灯、VIP |
| 危险 | `#FF4D4D` | 夜袭、城门危险 |
| 成功 | `#55E6A5` | 修复、补给、协作完成 |
| 弱文字 | `#9FB7C8` | 注释、空数据 |

字体建议：

- 中文使用当前项目字体：`AlibabaPuHuiTi-3-85-Bold SDF`。
- 标题粗体，正文中粗，数字更亮。
- 不要用花体，不要负字距。
- 小 UI 字号最低不要低于 18px，主榜单数字建议 22-28px。

材质建议：

- 面板是半透明磨砂冰晶玻璃，不是实心卡片。
- 边框用细线和局部冰裂纹，角落可以有少量晶体装饰。
- 避免全屏同一种蓝色，必须用金色、红色、绿色做功能性对比。
- 不要用大量渐变球、光斑、装饰 blob。

## 6. 关键状态效果图需求

至少跑 5 张效果图：

1. **标准 Running 状态**  
   白天或普通防守，顶部状态栏、游戏主体左上角悬浮资源贡献榜、右上角悬浮守护者榜、弹幕区都正常显示。

2. **守护者榜高光状态**  
   Top1 和一个聚光灯玩家被金色高亮，表现 `★ 玩家名`、贡献值发光、行背景扫光。

3. **夜晚危机状态**  
   顶部阶段变成夜晚/暴风雪，城门血量偏低，红橙警告脉冲，事件 toast 出现。

4. **礼物爆发状态**  
   中下区域出现礼物动画和 VIP 公告，守护者榜有排名变化闪光，但不遮挡主体战斗。

5. **建造投票/协作状态**  
   `BuildVotePanel` 和 `CoopMilestoneBar` 出现，表现观众一起投票建造或修复设施。

可选第 6 张：

- **低数据/开局状态**：榜单为空或只有 1-2 人，资源贡献榜使用 `—` 占位，整体不能显得坏掉。

```text
不要做成登录页、宣传海报、卡牌抽卡页或大标题落地页。不要让 UI 覆盖屏幕中心主体。不要使用过度紫色渐变、巨大光球、模糊暗背景、不可读小字、复杂花体、厚重圆角卡片堆叠。不要让所有面板都是同一种蓝色。不要生成英文占位 UI，主要文案要是中文。
```

## 8. Unity prefab 调整建议

1. 根节点继续保持全屏拉伸，默认 inactive 可以保留，由游戏流程打开。
2. 不要删除 `LiveRankingPanel`、`ResourceRankPanel` 和脚本序列化字段绑定，否则运行时会断。
3. `LiveRankingPanel` 建议后续单独拆成更干净的子 prefab，保留 5 个 `RankRow`。
4. `ResourceRankPanel` 保留食物、煤炭、矿石三类数据绑定，每类 3 个 TMP；竖屏施工时建议改成左上角三组竖向堆叠，不要临时改成动态生成，方便策划调样式。
5. 弹幕文案里仍有“雪球/阵营”旧玩法内容，效果图阶段建议统一替换为“补给/修门/守护/建造”等生存玩法文案。
6. 礼物、VIP、事件、投票、里程碑这些模块都可以继续保持嵌套 prefab，先做统一视觉语言，再逐个 prefab 微调。

## 9. 验收标准

- 1080 x 1920 下，顶部状态、守护者榜、资源榜、底部弹幕四大信息区都能同时读清。
- 中央至少 60% 视觉区域留给游戏画面。
- 守护者榜 Top5 任何一行都不会因为 6 字玩家名和 5 位数贡献值而挤压。
- 夜晚危机、礼物爆发、聚光灯三种高反馈状态有明显差异。
- 美术效果统一，但功能颜色明确：冰蓝=普通 HUD，金色=贡献/VIP，红橙=危险，绿色=修复/完成。
- 后续落回 Unity prefab 时，每个功能区都能独立调位置、底图、字体、动效，不需要在一个大 Canvas 里到处找零散物体。
