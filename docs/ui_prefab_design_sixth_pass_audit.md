## 2. 检查范围

- 每个资料包是否对应真实 prefab。
- 每个资料包是否不属于被父面板覆盖的跳过子 prefab。
- 每个 README 是否列出 6 个标准文件并引用正确 prefab 路径。
- 每个 UI plan 是否包含正确标题、prefab 路径和 1080x1920 画布信息。
- 每个 layout md 是否引用正确 SVG 并包含坐标表。
- 每个 AI prompt 是否引用正确 PNG 并包含标准章节。
- 每个 SVG 标题和尺寸是否与资料包一致。
- 每个 PNG 文件头尺寸是否为 1080x1920。
- 总索引是否链接二到五轮审计文件、全量矩阵和运行时绑定审计。
- 全量矩阵是否有 54 行 prefab 记录。
- 父面板子 prefab 附录是否覆盖所有直接嵌套子 prefab。

## 3. 发现问题

未发现交叉引用问题。

## 4. 结论

第六轮没有发现需要修复的错链、漏链、旧名或重复目录。当前资料之间的引用关系是闭合的：从 prefab 可以追到资料包或父面板附录，从资料包可以追到 prefab、SVG、PNG、AI prompt、README，从总索引可以追到所有审计报告。
