# AnnouncementUI_AnnouncementPanel UI 策划案

> Prefab：`Assets/Prefabs/UI/Panels/AnnouncementUI_AnnouncementPanel.prefab`  
> 用途：全屏公告层  
> 主效果图规格：1080 x 1920 竖屏  
> 日期：2026-05-07

## 1. 界面定位与目的

在战斗状态切换、失败保护、守护开始等关键节点给玩家和直播观众一个强提示。

该界面应被视为 全屏公告层，不是营销页。设计第一屏必须直接表达真实可用状态，并保留足够游戏主体安全区或明确的弹窗遮罩关系。

## 2. 设计目标

- 主标题 1 秒内可读
- 不遮挡太久，避免影响战斗观察
- 和其他 toast/跑马灯拉开层级

## 4. 推荐布局与层级

全屏透明遮罩上放置 MainText 与 SubText，短时淡入淡出，不承载长期交互。

## 5. 模块设计说明

### 暗化游戏背景

- 对应节点：`CanvasGroup`
- 建议位置：x=0 y=0 w=1080 h=1920。
- 设计要点：短时 alpha 0.45

### 主公告标题

- 对应节点：`MainText`
- 建议位置：x=120 y=690 w=840 h=118。
- 设计要点：例如：守护开始

### 副标题说明

- 对应节点：`SubText`
- 建议位置：x=170 y=820 w=740 h=72。
- 设计要点：例如：坚持到黎明！

### 淡出安全区

- 对应节点：`Animation`
- 建议位置：x=250 y=930 w=580 h=90。
- 设计要点：仅施工标注

## 6. 视觉风格

冰蓝深色遮罩，主标题用高亮白金色，副标题低饱和蓝白，动画以缩放和淡入为主。

建议保持项目统一语言：冰蓝代表常规 HUD，金色代表贡献/VIP/奖励，红橙代表危险或不可逆操作，绿色代表修复/完成。所有文字必须在 1080x1920 竖屏下可读，按钮文字不得溢出。

## 7. AI 效果图场景

守护开始、堡垒失守、新手保护等状态变化时，画面中央出现大字公告，背景游戏仍可辨认。

效果图应把施工原型里的坐标和灰盒理解为布局约束，而不是最终视觉元素。最终图可以加入材质、光效、图标、进度条和真实游戏背景，但不要出现 prefab 名、坐标、虚线框、施工标注等文字。

## 8. 实现注意事项

- CanvasGroup 控制整体 alpha；MainText/SubText 保持居中对齐。
- 显示时间建议 2-4 秒，和结算/暂停等 A 类弹窗互斥。
- 所有临时层应通过 CanvasGroup 或子节点显隐控制，避免长期遮挡中央 gameplay 区。
- 动态文本需要预留 20%-30% 的宽度余量，中文、数字和昵称都要能截断或滚动。
- 若该界面作为其他主面板的子面板复用，应优先服从主面板坐标，不再另起一套视觉规范。
