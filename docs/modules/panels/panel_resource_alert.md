# 面板规格：资源警报面板

> Canvas: Alert_Canvas (L_ALERT=30) | 基准: 1080×1920
> 引用: `docs/modules/ui_theme.md`
> 触发：当任一资源低于警告阈值，或失败条件即将触发时

---

## 面板总体结构

```
Alert_Canvas
├── ResourceWarning_Panel   (资源低阈值警告横幅，可同时3条)
├── GateWarning_Panel       (城门危急全屏警告)
├── TempWarning_Panel       (炉温极低全屏警告)
└── IntelligentTip_Panel    (智能指令提示横幅)
```

---

## 1. ResourceWarning_Panel：资源低阈值警告

| 属性 | 值 |
|------|-----|
| 位置 | 顶部HUD下方，Y=-130，紧贴HUD底边 |
| 尺寸 | 540×60px（左半屏） |
| 背景 | 半透明红色，C_DANGER_RED + 30%透明 |
| 最大同时3条 | 纵向堆叠，每条60px |

### 警告横幅格式
```
[资源图标 32×32] [资源名] 不足！[当前值/上限]  [建议指令: "发X采集"]
```

| 属性 | 值 |
|------|-----|
| 图标 | 对应资源图标，32×32px |
| 文字 | F_BODY(28px, C_DANGER_RED) |
| 动画 | A_SLIDE_UP 0.3s进入；A_FLASH_WARN 0.5s周期闪烁（背景） |
| 消失条件 | 资源恢复至警告阈值以上后淡出（A_TOAST_OUT） |

---

## 2. GateWarning_Panel：城门危急全屏警告

**触发条件**：城门HP ≤ 30%（300/1000）

| 属性 | 值 |
|------|-----|
| 背景 | 全屏边缘红色Vignette效果（四角暗红光晕） |
| 中心内容 | 无（不遮挡游戏画面） |
| 持续方式 | 只要城门HP≤30%就持续显示 |

**边缘红色Vignette实现**：
- 4个角落各一个Image，红色(`#FF2200`)，80%透明
- 向中心渐变（Gradient）
- A_FLASH_WARN 1s周期闪烁

**城门HP < 10% 额外效果**：
- 屏幕轻微持续震动（X±2, Y±2, 0.5s周期）
- 底部大字（半透明）："城门危急！发6打怪！"

---

## 3. TempWarning_Panel：炉温极低全屏警告

**触发条件**：炉温 ≤ -80℃

| 属性 | 值 |
|------|-----|
| 背景 | 全屏边缘蓝色Vignette（冰冻效果），四角冰蓝光晕 |
| 颜色 | C_COLD_PURPLE `#7766DD`，60%透明 |
| 持续 | 只要炉温 ≤ -80℃ 就持续 |
| 闪烁 | A_FLASH_WARN 0.5s周期（更快，更紧迫） |
| 提示文字 | "炉火将熄！发4生火！" F_H2(32px, C_COLD_BLUE)，屏幕底部 |

---

## 4. IntelligentTip_Panel：智能指令提示

**触发逻辑**：服务器根据当前资源状态，每15秒发送一次最紧迫资源提示

| 属性 | 值 |
|------|-----|
| 位置 | 底部弹幕区上方，Y=1440，居中 |
| 尺寸 | 640×72px |
| 背景 | C_BG_HUD（半透明深蓝） |
| 字体 | F_BODY(28px, C_TEXT_WHITE) |

**提示内容格式**：
```
💡 发"X" [资源名]不足，快来帮忙！
```

| 最紧迫状态 | 提示文字 |
|---------|---------|
| food < 100 | `💡 发"1" 食物不足，快来采鱼！` |
| coal < 60 | `💡 发"2" 煤炭不足，快来挖煤！` |
| furnace_temp < -50℃ | `💡 发"4" 炉火快灭了，快来生火！` |
| 夜晚+gate_hp < 50% | `💡 发"6" 城门受攻，快来打怪！` |

**动画**：A_SLIDE_UP 进入；显示5秒；A_TOAST_OUT 退出
