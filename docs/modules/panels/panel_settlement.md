# 面板规格：结算三屏

> Canvas: Settlement_Canvas (L_SETTLEMENT=50) | 基准: 1080×1920
> 引用: `docs/modules/ui_theme.md`
> 触发：游戏结束事件（food=0 / temp≤-100 / gate=0）

---

## 面板总体结构

```
Settlement_Canvas
├── ScreenA_Days        (结算A屏：存活天数，预创建，SetActive)
├── ScreenB_Stats       (结算B屏：数据统计，预创建，SetActive)
└── ScreenC_MVP         (结算C屏：MVP展示，预创建，SetActive)
```

**切换时序**：
1. ScreenA 出现（3秒）→ 渐出
2. ScreenB 出现（5秒）→ 渐出
3. ScreenC 出现（3秒）→ 渐出
4. 自动进入待机界面（bgm切回bgm_day_winter等待下局）

| 属性 | 值 |

## B屏：数据统计

| 属性 | 值 |
|------|-----|
| 背景 | 全屏，C_BG_PANEL |
| 标题 | "本局数据" F_H1(40px, C_TITLE)，顶部居中，Y=-120 |

### 统计项列表（6项，等间距排列）

| 统计项 | 图标 | 数值字体 | 说明 |
|--------|------|---------|------|
| 参与人数 | 👥 | F_NUM 48px | 本局发过弹幕的用户数 |
| 弹幕总数 | 💬 | F_NUM 48px | 有效弹幕指令总计 |
| 礼物总金额 | 💎 | F_NUM 48px, C_GOLD_ALERT | 以"抖币"为单位显示 |
| 击退怪物数 | 👾 | F_NUM 48px | 所有波次怪物合计 |
| 最高在线人数 | 📊 | F_NUM 48px | 同时在线观众峰值 |
| 最长连续666 | ⚡ | F_NUM 48px, C_GOLD_ALERT | 一次666触发的最多人数 |

**每项布局**：
- 图标 64×64px + 标签 F_SMALL(20px) + 数值 F_NUM(48px)
- 纵向排列，每项间距 60px
- 数值出现时用 A_NUM_ROLL(0.5s) 动画

## 过渡动画

| 过渡 | 动画 |
|------|------|
| A屏出现 | Alpha:0→1, 0.5s |
| A屏消失 | Alpha:1→0, 0.5s |
| B屏出现/消失 | 同上 |
| C屏出现 | Alpha:0→1, 0.5s；背景雪花ParticleSystem启动 |
| 返回待机 | 整个Settlement_Canvas Alpha:1→0, 1s |
