# 【提案评估】极地生存法则 - 提案内容

## 一、玩法题材

### 游戏概述
极地生存法则是一款抖音直播间互动生存游戏。观众通过刷礼物指挥矿工采集资源、升级防御，在极寒夜晚抵御怪物入侵，坚持到天亮即为胜利。

### 游戏题材
- **核心物象**：矿工角色（kuanggong系列，部落守卫者风格）、怪物、城门堡垒、矿场工位
- **故事背景**：极寒冬日，一群矿工被困在荒野堡垒中。白天必须采集食物、煤炭、矿石维持生存，夜晚怪物来袭时全员转入防御，守护城门。观众作为"指挥官"，通过刷礼物为矿工提供补给和增援，帮助他们坚持到天亮。
- **主题基调**：紧张、策略、团队协作、生存挑战

### 美术风格
- **角色美术**：矿工采用部落守卫者风格的3D骨骼动画角色（kuanggong系列），配有镐子等武器道具，Q版但不失力量感
- **怪物美术**：同系列3D角色，体型略小于矿工；Boss体型更大（约2.5m游戏高度），视觉压迫感强
- **场景设计**：冰雪荒野主题，中央堡垒+矿场工位+城门防线，冷色调为主（蓝/白/灰），营造极寒氛围
- **UI风格**：顶部资源栏（食物/煤炭/矿石/城门血量/温度）+ 右侧实时贡献榜，简洁实用

### 界面布局
- **游戏主界面**：顶部HUD资源栏 → 中央3D场景（堡垒+矿工+怪物） → 右侧实时排行榜Top5 → 底部礼物通知横幅
- **等待界面**：场景预览 + "等待主播开始"提示
- **结算界面**：三屏序列（A: 胜利/失败结果 → B: 资源统计数据 → C: Top3贡献者/MVP展示）

### 整体美术呈现
游戏采用冷色调冰雪场景搭配暖色UI元素，白天场景明亮（采集阶段），夜晚场景变暗（防御阶段），通过昼夜交替营造紧张感和节奏变化。矿工角色辨识度高，怪物入侵时画面紧张刺激，胜利结算时温暖喜悦。

## 三、系统设计

### 3.1 玩法框架

#### 游戏对局流程

```
【初始化 - Idle阶段】
├─ 主播开启游戏
├─ 服务器创建房间
└─ 等待观众加入

【加载 - Loading阶段】
├─ 同步游戏初始状态
├─ 初始化矿工池（最多12个矿工）
└─ 重置资源数值

【游戏进行 - Running阶段】
├─ 白天（Day Phase）
│   ├─ 观众刷礼物 → 分配/召唤矿工
│   ├─ 矿工前往工位采集资源（食物/煤炭/矿石）
│   ├─ 服务器推送资源产出更新
│   └─ 资源用于维持温度、修复城门
│
├─ 夜晚（Night Phase）
│   ├─ 所有矿工撤离工位，集结防守城门
│   ├─ 怪物波次生成（同屏上限15只）
│   ├─ 怪物优先攻击最近矿工，无矿工时攻城门
│   ├─ 矿工死亡后倒计时复活
│   └─ Boss怪物出现（体型更大，伤害更高）
│
└─ 昼夜循环，直到达成胜利/失败条件

【结算 - Settlement阶段】
├─ 屏幕A：胜利/失败结果展示
├─ 屏幕B：资源统计（采集量、击杀数等）
└─ 屏幕C：Top3贡献者 + MVP展示
```

#### 参与视角

**主播视角**：
- 完整3D游戏画面，包含所有矿工、怪物、资源变化
- 顶部HUD实时显示：食物、煤炭、矿石、城门血量、温度
- 可通过设置面板调整BGM/SFX音量

**参与观众视角**：
- 通过刷礼物参与游戏，礼物效果实时反馈
- 右侧实时贡献榜Top5（3秒刷新）显示自己的排名
- 礼物横幅通知展示效果描述
- 矿工头顶显示对应观众名字

**旁观观众视角**：
- 观看完整游戏画面和对局动态
- 实时排行榜更新
- 可随时刷礼物加入参与

#### 核心机制
1. **昼夜循环**：白天采集资源积累实力，夜晚防御消耗资源，形成策略张力
2. **资源管理**：食物（维持矿工体力）、煤炭（维持温度）、矿石（升级防御）、城门血量（生命线）
3. **服务器权威**：所有游戏状态由服务器计算和推送，客户端负责表现层
4. **矿工分配**：就近分配算法，矿工自动寻找最近空闲工位

#### 核心玩法设计要素

1. **昼夜循环系统**
   - 白天：矿工分散到工位采集，服务器定时推送资源产出
   - 夜晚：矿工集结防守，怪物波次入侵（普通怪+Boss）
   - 温度持续下降，需要煤炭维持

2. **矿工系统**
   - 最多12个矿工同屏（性能优化上限）
   - 矿工有生命值，夜晚显示血条
   - 死亡后倒计时自动复活
   - 头顶显示对应观众名字

3. **怪物系统**
   - 同屏最多15只怪物
   - 优先攻击最近矿工，无矿工时攻城门
   - Boss怪物体型更大，伤害更高
   - 波次递增难度

4. **资源与生存**
   - 食物：维持矿工体力
   - 煤炭：维持堡垒温度
   - 矿石：升级防御
   - 城门血量：生命线，归零则失败
   - 温度：过低会冻结矿工

### 4.3 其他补充信息

#### 技术实现
- **服务器权威架构**：Node.js WebSocket服务器，所有游戏逻辑服务器端计算，防作弊
- **Unity客户端**：Unity 2022.3 + URP渲染管线，3D骨骼动画角色
- **性能优化**：矿工上限12、怪物上限15、LOD裁剪、阴影关闭、Mesh压缩，目标20+ FPS
- **网络同步**：WebSocket实时通信，服务器推送资源/状态更新

#### 运营建议
- **难度分级**：可扩展休闲/困难/地狱三种难度模式
- **排行系统**：单局贡献榜 + 可扩展周榜/月榜
- **礼物梯度**：1~520抖币覆盖全价位观众，低门槛高上限
- **社交互动**：矿工头顶显示观众名字，增强归属感和参与动力

#### 预期数据
- **单局时长**：5-15分钟（根据难度）
- **适配观众规模**：10-5000人
- **礼物参与率**：预期5-15%观众参与刷礼物
- **核心卖点**：生存紧张感 + 团队协作 + 昼夜节奏变化 + 实时反馈
