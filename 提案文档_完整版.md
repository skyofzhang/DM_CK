# 【提案评估】卡皮巴拉对决 - 完整上架提案

## 一、玩法题材

### 游戏概述
卡皮巴拉对决是一款抖音直播间互动竞技游戏，采用双阵营对抗的创新玩法设计。

### 游戏题材
- **核心物象**：可爱萌趣的卡皮巴拉水豚角色与硕大的橘子
- **故事背景**：两支水豚阵营为争夺温泉的发言权而展开激烈对决
- **主题基调**：温暖、竞争、欢乐、互动

### 美术风格
- **角色美术**：卡皮巴拉和各类萌宠单位采用Q版卡通风格，可爱温暖
- **场景设计**：温泉主题的场景，线条柔和，色彩温暖（橙色/黄色系）
- **物象设计**：巨大橘子作为中心推动物体，符号化强、易辨识
- **UI设计风格**：扁平化现代设计，配色与主题一致，清晰易读

### 界面布局
- **主战斗界面**：顶部推力条/倒计时 → 中间橘子 → 两侧阵营信息展示
- **结算界面**：胜利公告 → MVP展示 → 双列排行榜 → 积分分配
- **排行榜界面**：实时贡献排名 → 玩家成长记录

### 整体美术呈现
游戏采用明亮温暖的配色，强调竞争与欢乐的平衡，通过萌宠角色和温泉主题营造放松娱乐的氛围，同时通过实时推力条和排行榜强化竞争感。

## 三、系统设计

### 3.1 玩法框架

#### 游戏对局流程

```
【初始化】
├─ 主播开启游戏
├─ 重置场景与单位
└─ 等待观众加入

【游戏进行阶段】
├─ 观众通过弹幕加入左/右阵营
├─ 观众刷礼物为阵营增加推力
├─ 实时推力差推动橘子
├─ 首次到达±25位置即为胜利

【结算阶段】
├─ 播放胜利公告（4秒）
├─ 等待过渡（5秒）
├─ 展示结算排行（包含：双列排名、贡献度、积分瓜分）
└─ 返回等待开局

【玩家参与视角】（直播间观众）
主播视角：
└─ 完整游戏画面 + 实时对局动态

旁观视角：
└─ 排行榜实时更新 + 礼物效果展示
```

#### 核心机制
1. **双阵营对抗**：左（香橙温泉）vs 右（柚子温泉）
2. **推力积累**：多种礼物对应不同推力值和持续时间
3. **实时同步**：服务器权威位置 + 客户端平滑预测
4. **胜利判定**：橘子首次到达±45位置（对应地图范围-45~+45）

### 3.2 指令设计

#### 加入阵营指令
```
左阵营加入指令（任选其一）：
- 输入数字：1
- 输入汉字：左
- 输入阵营名：香橙

右阵营加入指令（任选其一）：
- 输入数字：2
- 输入汉字：右
- 输入阵营名：柚子
```

#### 礼物系统指令
观众通过抖音礼物系统直接赠送，对应不同价位：

| 礼物名称 | 价格 | 推力值 | 召唤单位 | 持续时间 |
|---------|------|--------|---------|---------|
| 柚子 | 10抖币 | 343 | 奔跑水豚 | 30秒 |
| 甜甜圈 | 52抖币 | 808 | 甜甜圈战士 | 90秒 |
| 电池 | 99抖币 | 1415 | 电池骑士 | 120秒 |
| 泡泡 | 199抖币 | 2679 | 水豚之神 | 150秒 |
| 空投 | 520抖币 | 6988 | 荒野游侠 | 160秒 |

**升级规则**：每送出5个同类礼物，第5个自动触发升级版本（推力 ×2.3-3.0倍）

#### 特殊互动指令
- **点赞互动**：点赞按钮给阵营 +2 推力（3秒内有效）
- **评论666**：弹幕出现"666"给阵营 +3 推力（5秒内有效）

### 3.3 玩法设计图

#### 游戏界面布局（1080×1920）

**主战斗界面**：
```
┌─────────────────────────────────────────┐
│ 顶栏：左推力 | 倒计时 | 右推力 | 距离指示│
├─────────────────────────────────────────┤
│ 左阵营信息     │    橘子（中心）   │右阵营信息│
│ - 前3名玩家    │  自转/倾斜/晃动  │ - 前3名玩家
│ - 总贡献度     │  推力对抗动画    │ - 总贡献度
│                │  物理反馈(晃动) │
├─────────────────────────────────────────┤
│ 底部：实时礼物通知 + 玩家加入动画        │
└─────────────────────────────────────────┘
```

**结算界面**：
```
┌─────────────────────────────────────────┐
│ "恭喜 XXX 阵营获胜！" （大字标题）      │
├─────────────────────────────────────────┤
│ MVP 玩家展示（头像 + 贡献度）            │
├─────────────────────────────────────────┤
│ 排行榜（双列布局）                      │
│ 左：第1名 | 贡献 | ... | 第10名          │
│ 右：第1名 | 贡献 | ... | 第10名          │
├─────────────────────────────────────────┤
│ 积分池瓜分：奖励分配明细                │
├─────────────────────────────────────────┤
│ [返回大厅] 按钮                         │
└─────────────────────────────────────────┘
```

#### 核心玩法设计要素

1. **推力系统**
   - 弹幕加入：永久 +10 推力
   - 礼物加成：10-520抖币对应 343-6988 推力
   - Buff效果：可叠加、可升级

2. **橘子动画**
   - **移动**：客户端预测 + 服务器校正（平滑流畅）
   - **旋转**：自转(Y轴 20°/s×8倍) + 倾斜(Z轴 40°)
   - **晃动**：推力不平衡时左右摇晃（频率0.8Hz, 幅度0.0005）

3. **单位系统**
   - 随礼物动态召唤对应推力的萌宠单位
   - 单位排列成行，形成视觉压迫感
   - 支持升级（同类礼物5倍触发）

4. **反馈系统**
   - 实时推力条动画
   - 排行榜秒级更新
   - 单位弹入特效
   - 音效反馈（可选）
